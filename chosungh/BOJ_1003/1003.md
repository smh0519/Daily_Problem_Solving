# 1003 풀이방법 및 트러블슈팅

---

### 문제
다음 소스는 N번째 피보나치 수를 구하는 C++ 함수이다.

```cpp
int fibonacci(int n) {
    if (n == 0) {
        printf("0");
        return 0;
    } else if (n == 1) {
        printf("1");
        return 1;
    } else {
        return fibonacci(n‐1) + fibonacci(n‐2);
    }
}
```

fibonacci(3)을 호출하면 다음과 같은 일이 일어난다.

fibonacci(3)은 fibonacci(2)와 fibonacci(1) (첫 번째 호출)을 호출한다.

fibonacci(2)는 fibonacci(1) (두 번째 호출)과 fibonacci(0)을 호출한다.

두 번째 호출한 fibonacci(1)은 1을 출력하고 1을 리턴한다.

fibonacci(0)은 0을 출력하고, 0을 리턴한다.

fibonacci(2)는 fibonacci(1)과 fibonacci(0)의 결과를 얻고, 1을 리턴한다.

첫 번째 호출한 fibonacci(1)은 1을 출력하고, 1을 리턴한다.

fibonacci(3)은 fibonacci(2)와 fibonacci(1)의 결과를 얻고, 2를 리턴한다.

1은 2번 출력되고, 0은 1번 출력된다. N이 주어졌을 때, fibonacci(N)을 호출했을 때, 0과 1이 각각 몇 번 출력되는지 구하는 프로그램을 작성하시오.

--- 

### 입력
첫째 줄에 테스트 케이스의 개수 T가 주어진다.
각 테스트 케이스는 한 줄로 이루어져 있고, N이 주어진다. N은 40보다 작거나 같은 자연수 또는 0이다.

```bash
3
0
1
3
```

---

### 출력
각 테스트 케이스마다 0이 출력되는 횟수와 1이 출력되는 횟수를 공백으로 구분해서 출력한다.

```bash
1 0
0 1
1 2
```

--- 

### 풀이 전략
이 문제는 피보나치의 응용 문제로, 입력받는 N의 최대 크기가 40입니다. 이를 재귀호출로 해결할 경우 중복되는 연산이 있기에 시간 복잡도가 무려 2^n입니다. 따라서 불필요한 게산을 예방하고자 dp를 활용하여 풀었습니다. 

---

### 첫 번째 방식 풀이 과정

1. `fibonaci` 함수 내에 `dp`라는 이름의 배열을 생성하여 미리 0과 1을 호출할 때의 값들을 저장했습니다.

2. 이후 인자로 넘겨받은 수만큼 반복문을 실행해 각 수의 0과 1 호출 횟수를 계산하여 저장합니다.

3. 이후 입력한 수의 0과 1 호출 횟수를 출력합니다.

>#### 이 방식의 문제점
>이 풀이도 정답이었지만, 함수 호출 시마다 `dp`배열이 리셋되는 구조때문에 fibonaci(5)의 연산 결과가 fibonaci(4) 호출 시 활용되지 못했습니다. 함수 내부에서는 DP 로직이 맞을지라도, 전체 코드 관점에서는 중복 연산을 막지 못하므로 진정한 DP로 볼 수 없다는 생각이 들었습니다.

---

### 두 번째 방식 풀이 과정

1. `fibonaci` 함수 내에 있던 `dp`배열을 함수 밖으로 이동하였습니다.

2. 만약 인자로 넘긴 값이 `len(dp)`보다 작으면 이미 계산되었기 때문에 바로 출력했습니다.

3. 만약 인자로 넘긴 값이 `len(dp)`보다 크면 계산을 하도록 수정했습니다.

---

### 시간 복잡도

- 첫 번째 방식: O(n * m)
- 두 번째 방식: O(n + m)
